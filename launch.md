### 💱 1.2 Vora Swap (`voraswap.com`)
- [o] 스왑(Swap) 인터페이스의 토큰 선택 및 수량 입력 버그 확인
- [o] 슬리피지(Slippage) 설정 및 예상 수령액 계산 로직 점검
- [x] 스왑 실행 버튼 클릭 시 트랜잭션 정상 생성 및 결과 안내(성공/실패) 모달 확인
: 스왑 시 보라 중개 와 인출 거래를 나눠서 관리할 필요가 있습니다. 같이 있어서 유저들이 혼란을 일으킬 수 있습니다.

: p2p 마이 오피스에서 체결한 보라를 업로드 했는데, p2p에 반영이 안되는 거 같아요. 이제는 실전에서 에스크로와 락업의 개념을 적용해야 하니, p2p 승인 정상 작동 되게 해야 합니다. p2p에서 거래된 내역은 p2p 전용 풀에 저장하고, 관리자에서 승인하면, 내역이 전송되도록 해야 합니다.
  : 관리자 대시보드 유저 관리 패널 추가 (가장 중요한 기능)
  : 브로모션 드라이브 영업 추천 링크는 관리자에서 링크 계정 관리 될 수 있게 해줘야 합니다. 불법적인 p2p : 자금 세탁 방지를 위해 kyc 인증 거래 시스템 구축 필요 합니다.
  : 계정을 추천 링크를 잠시 정지 시킬수 있는 기능도 필요 합니다. 정지 사유는 개별 전달 하고, 정지 시 유저가 로그인 하면 사유를 팝업 형태로 전달 해야 합니다.
  : 추천인과 추천 대상자 모두에게 보상을 지급 하는지, 아니면 추천인에게만 보상을 지급 하는지 확인 하고, 봇이 작동 가능 한지 확인 해야 합니다.
