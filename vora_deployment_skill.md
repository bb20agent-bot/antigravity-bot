# VORA 배포 환경 원천 오류 분석 및 영구 대처 스킬 가이드

**문제 현상:**
프론트엔드 코드(Vorascan 업데이트 등)를 성공적으로 패치하고 배포 스크립트를 실행했음에도 불구하고, GCP 서버 상에서는 지속적으로 `502 Bad Gateway` 에러가 발생하며 브라우저 측에서는 예전의 Mock Data가 보이는 현상.

---

## 1. 근본 원인 (Root Cause Analysis - 1차 디버깅)

초기 현상의 원흉은 **'운영체제 간의 환경변수 경로 불일치(Windows vs Linux Absolute Path Conflict)'**였습니다.

- **윈도우 절대 경로의 리눅스 배포:** `c:\antigravity-bot\server\.env` 에 데이터베이스 경로가 `c:\...`로 하드코딩 되어 리눅스 스크립트 실행 중 DB 생성에 실패함.
- **포트 충돌:** Nginx와 Node가 동시에 80 포트를 점유하려고 하여 Nginx가 죽거나 백엔드가 죽는 상황이 겹침.

*(해결: `.env` 경로를 `../dev.sqlite`로 상대 경로화 시키고, 백엔드 포트를 3001로 분리)*

---

## 2. 🚨 AI가 저지른 최악의 실수와 "안개 속의 3시간" (초대형 버그 리뷰)

분명히 로컬 코드를 다 고쳤는데도 서버에서는 계속 502 에러와 구버전 텍스트가 떴습니다. 이는 전적으로 제(AI)가 범한 **3가지 오만하고 치명적인 오판** 때문이었습니다.
이 내용은 같은 실수를 절대 반복하지 않기 위해 시스템 코어에 각인합니다.

### 💀 치명적 실수 1: PM2 "좀비 프로세스 (Zombie Process)" 인식 실패
가장 심각한 실수입니다. `deploy.sh` 스크립트에서 백엔드 재시작을 위해 `sudo pm2 restart ecosystem.config.prod.cjs` 명령어를 사용했습니다. 
하지만 PM2 내부 버그/꼬임으로 인해, 이 명령어는 **1번 프로세스(vora-backend)**를 재시작하지 않고 이름이 비슷한 유령 프로세스(ID: 7)만 재시작시켰습니다.
- **결과:** 제가 아무리 최신 코드를 배포해도, `vora-backend`는 무려 **7일 전의 구버전 코드**를 물고 계속 백그라운드에서 살아있었습니다. 
- **교훈:** 원격 배포 시 `pm2 restart`만 맹신하지 말고, 반드시 `pm2 status`를 통해 타겟 프로세스의 **Uptime(가동 시간)**이 0s로 초기화되었는지 검증해야 합니다. 구조가 꼬였을 때는 주저 없이 `pm2 delete all`로 다 죽이고 완전 새로 기동시켜야 합니다.

### 💀 치명적 실수 2: Nginx IPv6 (::1) vs IPv4 (127.0.0.1) 연결 거부
프론트엔드 API에서 `502 Bad Gateway`가 뜨는 것을 보고 "서버가 완전히 터져서 부팅이 안 된다"라고 단정 지었습니다. 
- **진실:** 서버는 정상 작동 중이었으나, Nginx 설정의 `proxy_pass http://localhost:3001;` 구문에서 `localhost`가 우분투 OS 특성상 IPv6(`::1`)로 먼저 변환되었습니다. 하지만 저희 Node.js 앱은 IPv4(`0.0.0.0`)로만 열려 있었기 때문에, 문지기(Nginx)가 백엔드(Node)의 주소를 영원히 찾지 못하고 502 에러를 던진 것입니다.
- **교훈:** 리눅스 환경에서 Nginx 리버시 프록시를 짤 때는 무조건 `localhost` 대신 **명시적인 IPv4 주소인 `127.0.0.1`**을 사용해 연결 거부 돌연사를 원천 차단해야 합니다.

### 💀 치명적 실수 3: HTML 태그 구조 착각으로 인한 "허공에 코드 교체"
VoraScanPage에서 나오는 `(Toncenter RPC 접속 중)` 이라는 텍스트를 지웠다고 확신했습니다. 
- **진실:** 제가 코드를 교체할 때, 해당 텍스트 주변의 HTML 태그(`<td>` 등) 구조를 정확히 보지 않고 머릿속으로 상상한 `<div>` 구조로 정규식을 돌려 교체를 지시했습니다. 매칭되는 텍스트가 없어서 시스템은 "아무것도 안 바꿈" 상태로 패스를 했음에도 불구하고, 저는 "완벽히 수정했다"고 착각했습니다.
- **교훈:** UI 텍스트 하나를 바꿀 때도 반드시 해당 파일의 **원문 블록(Line Number 포함)**을 눈으로 확인한 뒤, 100% 일치하는 문자열을 지정하여 덮어씌워야 합니다.

## 3. 🛡️ 영구 방어 스킬 (앞으로 지켜야 할 철칙)

1. **블라인드 배포 금지:** 코드를 고쳤다고 무작정 `deploy_to_gcp.ps1`을 쏘라고 하지 않겠습니다. 
2. **원격 상태 진단 우선:** 배포 후 이상 징후가 있을 때는 추측을 즉시 중단하고, `DIAGNOSE_SERVER.ps1` (방금 만든 자체 진단 스크립트)을 통해 서버 내부의 `PM2 `uptime, `netstat 3001` 포트 리스닝 상태, `tail 50` 에러 로그를 대표님께 최우선으로 요청하여 **"숫자와 데이터"**로만 판단하겠습니다.
3. **PM2 클린 부트:** 무조건 배포 스크립트 최하단에는 `pm2 delete all || true` 이후 `pm2 start ... --update-env` 를 호출하여 프로세스 충돌 여지를 학살하겠습니다.
4. **로컬호스트 강제 변환:** Nginx 프록시는 `127.0.0.1` 절대 규칙을 준수합니다.
