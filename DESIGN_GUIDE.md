# VORA AI: Project Design & Architecture Guide

## 1. Project Overview
**Project Name:** VORA AI (Anti-Gravity Bot Platform)
**Vision:** A gamified, futuristic trading platform combining AI-driven trading strategies with user engagement (Tap-to-Earn), NFT identity, and decentralized governance.
**Core Concept:** "Automating Your Assets" - Empowering users with high-leverage tools and automated strategies in a premium, mobile-first interface.

## 2. Visual Design System
The user interface is designed to evoke a "Premium Futuristic" aesthetic, utilizing dark themes, glassmorphism, and vibrant neon accents to create an immersive experience.

### 2.1 Color Palette
*   **Backgrounds:**
    *   `#050505` (Deep Void Black) - Main background.
    *   `rgba(255, 255, 255, 0.05)` - Glass cards / Panels.
*   **Primary Brands:**
    *   **VORA Blue:** `#0088cc` (Used for active states, branding, primary actions).
    *   **Force Purple:** `#a855f7` (Used for secondary accents, glows, premium tiers).
*   **Functional Colors:**
    *   **Success:** `#22c55e` (Green-400/500) - Profits, active status, upward trends.
    *   **Warning:** `#ef4444` (Red-500) - Downward trends, locked states.
    *   **Gold:** `Yellow-500` - Ranks, legendary items.

### 2.2 Typography & Elements
*   **Headings:** Uppercase, Bold/Black weight, Wide tracking (`tracking-widest`).
*   **Numbers:** Monospace / Tabular nums for financial data.
*   **Icons:** Lucide-React library (Thin strokes, unified visual language).
*   **Components:**
    *   **Glass Cards:** `backdrop-blur-2xl`, thin white borders (`border-white/10`), inner glow.
    *   **Buttons:** Rounded pills or circles, distinct active/inactive states with scale animations (`active:scale-95`).
    *   **Animations:** Smooth fade-ins (`animate-in`), slide transitions, and "breathing" glows.

## 3. Application Architecture (Frontend)

### 3.1 Tech Stack
*   **Framework:** React 18 + Vite
*   **Language:** TypeScript
*   **Styling:** Tailwind CSS (Utility-first)
*   **State Management:** React Hooks (`useState`, `useEffect`)

### 3.2 Navigation Structure
The app uses a persistent bottom navigation bar with 4 core tabs:
1.  **HOME:** Dashboard, Portfolio, Signal Feed.
2.  **MY OFFICE:** Gamification, Referrals, Earnings.
3.  **MARKETPLACE:** NFT Trading, Digital Identity.
4.  **INFO:** Governance, Socials, Documentation.

## 4. Feature Modules

### 4.1 Dashboard (Home)
*   **User Profile:** Displays Rank (e.g., "GOLD ELITE"), UID, and Wallet Connection status.
*   **Portfolio Overview:**
    *   Real-time display of TON (Layer 1 token) and VORA (Platform token).
    *   Subscription management interaction (Unlock Logic).
*   **Leverage Terminal:**
    *   Quick-switch leverage options (3x, 5x, 10x) with tier requirements.
*   **Live Alpha Feed:**
    *   Vertical video/card feed of active trading strategies.
    *   Metrics: ROI %, Win Rate, Trader Name.
    *   **Locked Content:** Strategies are blurred/locked for non-subscribers.

### 4.2 Gamification (My Office)
*   **Tap-to-Earn (T2E):**
    *   Daily interaction mechanism to earn VORA tokens.
    *   Countdown timer (24h cycle) and visual cooldown.
*   **Referral System:**
    *   Tiered affiliate structure (Level 1-3).
    *   Copy-paste referral link generation.
*   **Network Stats:** Visual breakdown of user's network performance.

### 4.3 NFT Marketplace
*   **DNFT (Dynamic NFT) Gallery:**
    *   Trading characters (e.g., "Turtle Guru", "Iljimae").
    *   **Tiers:** Legendary, Epic, Rare.
    *   **Live Market:** Real-time price simulation with trend indicators (Up/Down arrows).

### 4.4 Governance (Info)
*   **Voting System:**
    *   Community proposals (e.g., "Implement Auto-Rebalancing").
    *   Visual progress bars for vote tallies.
*   **Social Hub:** Direct links to Telegram and X (Twitter).

## 5. Integration Architecture

### 5.1 Signal System
*   **Source:** Python / External Algo (`test_signal.py`).
*   **Transport:** HTTP POST (Webhook).
*   **Destination:** n8n Workflow Automation -> Frontend Update.
*   **Data Payload Structure:**
    ```json
    {
      "symbol": "BTCUSDT",
      "side": "BUY", 
      "strategy": "Anti-Gravity_V1",
      "reason": "Technical Indicator (e.g., Bollinger Band)"
    }
    ```

### 5.2 Data Models (Types)
*   `UserAssets`: Tracks TON and VORA balances.
*   `StrategyInfo`: Schema for strategy feed items (ROI, WinRate).
*   `DNFTCharacter`: Schema for marketplace items including price history trends.
