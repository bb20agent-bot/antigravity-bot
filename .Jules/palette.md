## 2024-05-07 - Add visually obvious copy feedback and a11y labels to copy button
**Learning:** System-level toast notifications are unreliable in Telegram Mini Apps, and standard clipboard API (`navigator.clipboard`) doesn't always work dependably in these restricted embedded contexts. As an app-specific accessibility constraint, we must always provide fallback clipboard mechanisms and inline visual feedback (e.g. icon swapping rather than just toasts) to guarantee the user is aware a copy action succeeded.
**Action:** When adding any copy-to-clipboard functionality to the Telegram Mini App frontend, always implement a `document.execCommand('copy')` fallback via a dynamically created `textarea`, and explicitly provide an inline icon change and dynamic `aria-label` updates on the button itself.
