## 2024-05-10 - Telegram Mini App Clipboard Fallback
**Learning:** System toasts and standard `navigator.clipboard.writeText` are unreliable in Telegram Mini Apps due to permission restrictions or lack of secure context. When implementing clipboard copy functionality, a fallback mechanism using `document.execCommand('copy')` with a temporary dynamically created `textarea` element is required. Because system-level toast notifications are unreliable, it is critical to explicitly provide inline visual feedback (e.g., swapping a Copy icon to a Check icon) and dynamically update `aria-label` attributes to communicate success to the user.
**Action:** Always implement the `textarea` + `document.execCommand('copy')` fallback for copy actions in this app, and ensure visual and accessible feedback (e.g., icon swap and `aria-label` update) is provided directly on the interaction element.
